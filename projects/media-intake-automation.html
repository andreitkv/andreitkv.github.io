The problem

Getting a reference was only the first half of the job.

Media arrived in different containers, codecs and resolutions. A successful transfer could still leave me with a file that needed another manual conversion before it was useful in an editing timeline.

I combined source handling, fallback logic, file validation and format preparation into one Finder-friendly action. The result is either a usable MP4 or a clear error — not a half-finished download.

One link.
One finished file.
No cleanup ritual.

The workflow

Built around failure handling, not only the happy path.

01

Preflight

The action checks its local tools and rejects empty or unsuitable input before starting a long operation.

  • Runtime checks
  • Single-item guard
  • Source validation
  • Clear notifications
02

Format fallbacks

Several format strategies are tried in a controlled order so a changed source does not immediately break the workflow.

  • Video + audio variants
  • Container merging
  • Output path capture
  • File existence checks
03

Editor preparation

FFprobe checks the result. Files outside the expected H.264 and AAC combination are converted automatically.

  • Codec inspection
  • 1080p scaling
  • H.264 video
  • AAC audio

Design decisions

A personal tool with an intentionally narrow public scope.

Finder first

The workflow runs as a macOS Quick Action instead of asking for repeated Terminal commands.

One item at a time

Playlist-style input is deliberately avoided. Each run produces one predictable media file.

Useful output

Success means the resulting file exists, is non-empty and is ready for the next production step.

Responsible use

The workflow is intended only for media I own or have permission to download and process.

Availability

This is a private workflow case study, not a public downloading service. Source code, account data and access credentials are not distributed through this portfolio.
